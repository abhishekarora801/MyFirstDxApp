# Firefly Image Studio

An Adobe Experience Cloud app — built on **Adobe App Builder** — that lets creative and marketing teams generate AI images with **Adobe Firefly** and save them directly to **AEM Assets** in a single click, without leaving Experience Cloud.

## Overview

The app collapses a multi-tool workflow (generate elsewhere → download → re-upload to AEM → organize) into one screen. Users describe an image in plain English, pick creative controls, generate via Firefly v3, preview the result, and push it straight into the DAM under a governed path.

## Features

- **Natural-language prompt** with Ctrl+Enter shortcut.
- **Creative controls** — content class (Photo / Art), aspect ratio (1:1, 4:3, 3:4, 16:9), style presets (Cinematic, Vintage, Anime, Watercolor, 3D Render, and more).
- **Live preview grid** with seed IDs, download links, and hover overlays.
- **One-click Save to AEM** — uploads to `/content/dam/firefly-generated` with auto folder creation and a direct asset link on success.
- **Native Experience Cloud integration** — runs inside the ExcShell with automatic SSO.

## Architecture

The app is an ExcShell extension with a React frontend and two single-responsibility serverless actions on Adobe I/O Runtime.

```
[React UI / ExcShell]
   |
   +-- action: firefly-generate  --> IMS OAuth S2S  --> Firefly v3 API
   |
   +-- action: aem-upload        --> IMS JWT exchange --> AEM Direct Binary Upload
                                                          (initiate -> PUT -> complete)
```

**Actions**

| Action | Purpose | Auth |
|--------|---------|------|
| `firefly-generate` | Calls Firefly v3 to create images from a prompt + controls | OAuth S2S (client_credentials) |
| `aem-upload` | Signs a JWT, exchanges for an AEM token, streams the image into AEM's Direct Binary Upload pipeline | JWT → IMS exchange (`ent_aem_cloud_api` metascope) |

Credentials are injected as action inputs via `ext.config.yaml` — nothing sensitive ever reaches the browser.

## Tech Stack

- **Frontend:** React 16, React Router (HashRouter), Adobe React Spectrum, `@adobe/exc-app`
- **Backend:** Adobe I/O Runtime (Node.js 22), `@adobe/aio-sdk`, `node-fetch`
- **Auth:** Adobe IMS — OAuth S2S for Firefly, JWT exchange for AEM
- **APIs:** Firefly v3 Generate, AEM Assets Direct Binary Upload

## Prerequisites

- Node.js ≥ 18 and the [AIO CLI](https://developer.adobe.com/app-builder/docs/getting_started/)
- An Adobe Developer Console project with **App Builder** and **Firefly API** access
- An **AEM Cloud Manager Integration** (technical account) with the `ent_aem_cloud_api` metascope and appropriate DAM permissions

## Setup

1. Clone the repo and install dependencies:
   ```bash
   npm install
   ```
2. Generate the `.aio` and `.env` files by pointing at your Developer Console project:
   ```bash
   aio app use
   ```
3. Populate `.env` with the values listed in [Config](#config) below.

## Local Dev

- `aio app run` — start the local dev server (UI on `localhost:9080`). Actions are deployed to Adobe I/O Runtime.
- `aio app dev` — run actions locally too.

See [aio app dev vs aio app run](https://developer.adobe.com/app-builder/docs/guides/development/#aio-app-dev-vs-aio-app-run).

## Test & Coverage

- `aio app test` — unit tests for UI and actions
- `aio app test --e2e` — e2e tests

## Deploy & Cleanup

- `aio app deploy` — build and deploy actions to Runtime and static files to CDN
- `aio app undeploy` — tear down the deployment

## Config

### `.env`

This file must **not** be committed to source control.

```bash
# Adobe I/O Runtime (generated by `aio app use`)
AIO_RUNTIME_AUTH=
AIO_RUNTIME_NAMESPACE=

# Firefly (OAuth S2S)
FIREFLY_CLIENT_ID=
FIREFLY_CLIENT_SECRET=
FIREFLY_SCOPES=openid,AdobeID,firefly_api,ff_apis
FIREFLY_API_URL=https://firefly-api-enterprise-stage.adobe.io/
IMS_TOKEN_URL=https://ims-na1-stg1.adobelogin.com/ims/token/v3

# AEM Cloud Manager Integration (JWT)
AEM_HOST=https://author-pXXXX-eYYYY.adobeaemcloud.com
AEM_CLIENT_ID=
AEM_CLIENT_SECRET=
AEM_TECHNICAL_ACCOUNT_ID=
AEM_ORG_ID=
AEM_METASCOPE=ent_aem_cloud_api
AEM_IMS_HOST=ims-na1.adobelogin.com
# Paste the PEM directly (with real newlines) or a single-line base64-encoded PEM
AEM_PRIVATE_KEY=
```

### `app.config.yaml`

The main configuration file that defines the application's implementation. The actions and their env bindings are declared in `src/dx-excshell-1/ext.config.yaml`.

See [configuration docs](https://developer.adobe.com/app-builder/docs/guides/configuration/#appconfigyaml) for more.

#### Action Dependencies

You have two options to resolve actions' dependencies:

1. **Packaged action file** — add dependencies to the root `package.json` and install them with `npm install`. Set `function` in `app.config.yaml` to the **entry file**. Webpack bundles the action into a single minified JS file. Use this to reduce action size.
2. **Zipped action folder** — add a `package.json` inside the action folder. Set `function` to the **folder**. Dependencies are installed locally and the folder is zipped. Use this to keep an action's dependencies isolated.
